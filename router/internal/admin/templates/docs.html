OpenAI-compatible endpoint
Drop-in replacement for api.openai.com
POST https://llm.teoh.co/v1/chat/completions
Authentication
Authorization: Bearer sk-yourkey
Example request
curl https://llm.teoh.co/v1/chat/completions \ -H "Authorization: Bearer sk-yourkey" \ -H "Content-Type: application/json" \ -d '{"model":"llama3.2:3b","messages":[{"role":"user","content":"hi"}],"stream":true}'
Python SDK
from openai import OpenAI client = OpenAI(base_url="https://llm.teoh.co/v1", api_key="sk-yourkey")
Anthropic-compatible endpoint
Drop-in replacement for api.anthropic.com
POST https://llm.teoh.co/v1/messages
Authentication
x-api-key: sk-yourkey
Python SDK
import anthropic client = anthropic.Anthropic(base_url="https://llm.teoh.co", api_key="sk-yourkey")
Responses API endpoint
OpenAI Responses API format
POST https://llm.teoh.co/v1/responses
Authentication
Authorization: Bearer sk-yourkey
Router config
config.yaml — read-only at startup
server: port: 53002

API keys and client tokens are managed through this UI and stored in state.json.

Client setup
client/config.yaml
router_url: "wss://llm.teoh.co/ws/client" router_token: "ct-yourname-yourtoken" max_concurrent: 4 models: - name: "llama3.2:3b" endpoint: "http://host.docker.internal:8080"
Docker deploy
Router (server) and client (local machine)
# Router docker compose up -d # Client (local machine with llama.cpp running on port 8080) docker compose -f docker-compose.client.yml up -d
Priority tiers
Requests are ordered by API key priority, then FIFO within each tier
high → processed first (0) normal → default (1) low → processed last (2)

Set priority per API key on the API Keys page.
