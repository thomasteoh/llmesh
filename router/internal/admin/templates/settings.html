{{/* ── Optimization tab ── */}}
Request optimization

Transformations applied to inbound requests. All default off; each is independent. None change what the model sees beyond the stated effect.

Canonicalize JSON (sort keys, drop insignificant whitespace, trim string content) when hashing for dedup, so semantically identical requests coalesce. Affects the dedup hash only; the dispatched request is unchanged.

Route follow-up turns of a conversation to the client that served the prefix, keeping the upstream KV cache warm. Falls back to normal load-spreading when the prior client is busy or cold.

Trim surrounding whitespace from plain-text messages and drop empty messages. Structured (tool/multimodal) content is left untouched; a request is never emptied entirely.

Also collapse runs of interior whitespace within each line (newlines preserved). Only takes effect when "Clean requests" is on.

Clamp temperature to [0, 2] and top_p to [0, 1]. A value of 0 is treated as unset and left alone.
